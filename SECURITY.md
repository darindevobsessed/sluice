# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Gold Miner, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

Instead, please use one of the following methods:

1. **GitHub Private Vulnerability Reporting** (preferred): Use GitHub's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) feature on this repository.

2. **Email**: Send details to [INSERT EMAIL]

## What to Include

When reporting a vulnerability, please include:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Suggested fix (if you have one)

## Response Timeline

- **Acknowledgment**: Within 48 hours of receipt
- **Initial Assessment**: Within 5 business days
- **Resolution Target**: Within 30 days for critical issues

## Scope

This security policy covers:

- The Gold Miner application code
- Dependencies used by the project
- Deployment configurations provided in this repository

## Process

1. **Report** — Submit vulnerability details via the methods above
2. **Acknowledge** — We confirm receipt within 48 hours
3. **Investigate** — We assess the severity and impact
4. **Fix** — We develop and test a fix
5. **Disclose** — We coordinate disclosure with the reporter

## Out of Scope

- Issues in third-party services (YouTube API, Anthropic API)
- Social engineering attacks
- Denial of service attacks

Thank you for helping keep Gold Miner secure.
